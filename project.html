What is JK Gaming and how does it work?

Within the gaming world there exists a circular problem that continues to hold back innovative new games. This is the problem that players play the games that provide the biggest payouts but the biggest payouts need the players to build out the prize fund.

Players VS Game Makers

Introduce Liquidity Providers

Winning Rating 50

Liquidity Providers (LPs) are presented a pool list through an interface similar to the Balancer and Symmetric AMM platform. This allows LPs to view a list of games along with metadata associated with each game including projected returns and how long the game runs over.

Upon choosing which games to provide liquidity to, LPs add liquidity for the duration of the game, in exchange for a percentage stake in the pools returns. This ensures that the game has a prize fund to attract players. Players then buy tokenised tickets in the game with ticket sales going into the game pool alongside the seed liquidity. The game is run within the pool contract for full transparency with the payout being split between LPs, players and JK Gaming. LPs get their stake plus interest back, players get the prize portion and JK Gaming gets 3% of the pool total to fund conntinued development and support of the platform.

The exact split is defined by the game maker with the majority of liquidity going back to players and liquidity providers.

For example, a game played over 10 days is created that uses a random number generator (RNG) at a fixed date and time with the closest player ticket number receiving the payout. LP's provide $1 million in seed funding which enncourages players to buy $4 millionn in tickets. The RNG runs and a winner is chosen.

For this particular game, the logic is clearly set out to give LPs 40% of the pool ($2 million), players 57% ($2.85 million) and JK Gaminng 3% ($150,000). LP's got their stake back plus an additional 100% over 10 days without needing to introduce any kind of new token while players were paid a significant $2.85 million jackpot.

Betting and gambling process

For LPs, the process is no different to investing in other Automated Market Maker (AMM) pool types. LPs look down the list of pools for a good return and add liquidity to the pool. Exchange pools can't charge users high fees so the APRs tend to be low, forcing projects to create their own tokens to try to create a more interesting return.

This isn't a problem for gaming pools which allows much higher returns to be set without creating new tokens. The LPs are paid first to remove any risk of loss from a game while ensuring a prize fund exists for players from the start.

The pool exists for the duration of the game with the game logic running to calculate the final split and payout, transferring funds automatically to players and LPs. Simple, transparent, fair, fast.

  • Liquidity Providers make big returns

    Games support much higher APRs that other DeFi pool types and so is set to become a very popular investment type for LPs.

  • Players win the big prizes

    High levels of seed funding by LPs provide prizes that attract players in the numbers needed to create the biggest prizes as well as best returns for LPs

  • Games Makers get funded

    Game makers are free to design the games they want to run, set out the game rules and logic and attract the liquidity and players needed to get going

Initially the focus is on simple games such as dice, lotteries and raffles but more complicated pool contracts will be developed over time including contracts that use oracles for sport and eSports as well as casino style gaminng.
